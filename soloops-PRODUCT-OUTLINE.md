## SAMPLE CHAPTER: Content Production System

*(Full 1,500-word chapter included in playbook)*

### The Problem

Most solopreneurs treat content like a craft. They sit down, feel inspired, write for 3 hours, edit for 2 more, format for another hour, then publish. One post = 6 hours. Two posts/week = 12 hours. That's 25% of your work week on content alone.

Doesn't scale.

### The System

My content production system publishes 6 posts/day across two blogs (workless.build + CleverDogMethod). Total time: 3 hours/week. Here's how.

**Step 1: Idea Queue (Notion)**

I don't write on demand. I write when ideas are cheap (random moments of clarity). Those go into a Notion database with three fields:
- Title (working headline)
- Angle (one-sentence unique take)
- Keywords (SEO targets)

No body text. No drafts. Just capturing ideas fast. Takes 60 seconds per idea. I aim for 50 ideas in the queue at all times.

**Step 2: Automated Drafting (OpenClaw + ChatGPT)**

A cron job runs twice daily (08:00 + 20:00 UTC). It:
1. Pulls the oldest "ready" idea from Notion
2. Sends to OpenClaw with this prompt:

```
Write 1,200-word blog post:
Title: {title}
Angle: {angle}
Keyword: {keyword}
Voice: Direct, specific numbers, anti-hustle
Include: Real examples, no filler
```

3. Saves output to `/drafts/YYYY-MM-DD-slug.md`

The AI does the first draft. Not perfect. But 70-80% there. Takes 90 seconds (API time).

**Step 3: Humanization (ai-humanizer skill)**

Before I see it, the draft runs through the ai-humanizer skill. This strips:
- "Delve into"
- "It's important to note"
- "Leverage"
- 22 other AI patterns I've trained it to catch

Output goes to `/ready/YYYY-MM-DD-slug.md`. Takes 10 seconds.

**Step 4: My Review (10 minutes/post)**

I review posts in batches (15-20 at a time). Friday mornings, usually. I'm looking for:
- Tone mismatch (does it sound like me?)
- Factual errors (did it make something up?)
- Weak examples (did it go generic?)

If it passes: Approve. If not: Edit for 5 minutes or delete.

Approval rate: ~80%. Meaning 4 out of 5 posts need zero edits. The other 1 needs minor tweaks.

**Step 5: Automated Publishing (GitHub + Vercel)**

Approved posts commit to GitHub:
```bash
git add posts/YYYY-MM-DD-slug.md
git commit -m "Post: Title Here"
git push origin main
```

Vercel detects the push, rebuilds the site, deploys. Takes 90 seconds. Post is live.

**The Math:**
- Manual: 6 hours/post × 6 posts/day = 36 hours/day (impossible)
- Automated: 10 minutes review/post × 42 posts/week = 7 hours/week
- With batching: 3 hours/week (review Friday, Monday)

**What I'm NOT doing:**
- Writing first drafts (AI does it)
- Formatting (Markdown + CI handles it)
- Publishing manually (Git push triggers deploy)
- Social promotion (SEO traffic replaces it)

**The Code (Included in Playbook):**

```python
# generate-post.py
import openai
import requests
from datetime import datetime

def generate_post(title, angle, keyword):
    prompt = f"""
    Write 1,200-word blog post:
    Title: {title}
    Angle: {angle}
    Keyword: {keyword}
    Voice: Direct, specific numbers, anti-hustle
    """
    
    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}]
    )
    
    content = response.choices[0].message.content
    
    # Save to file
    slug = title.lower().replace(" ", "-")
    filename = f"drafts/{datetime.now().strftime('%Y-%m-%d')}-{slug}.md"
    
    with open(filename, 'w') as f:
        f.write(content)
    
    return filename

# Run from cron: python3 generate-post.py
```

**The Cron Job:**

```bash
# /etc/crontab
0 8,20 * * * /usr/bin/python3 /root/scripts/generate-post.py
```

**Customization:**
- Change the prompt (your voice, your angle)
- Adjust frequency (1/day? 10/day? Your choice)
- Add human review gate (nothing publishes without approval)

**What This Unlocks:**

With content production automated, I can:
- Run 2 blogs with the same system (marginal cost: ~$10/month for domain)
- Focus on strategy, not execution (3h/week vs 30h/week)
- Test content ideas fast (50 ideas → 50 posts in 10 days if needed)

The constraint shifts from "time to write" to "ideas worth testing." That's a better problem.

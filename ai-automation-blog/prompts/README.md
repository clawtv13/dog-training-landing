# Prompts Directory

This directory contains prompts for blog post generation and A/B testing.

## 📂 Structure

```
prompts/
├── blog-generation.txt       # Current production prompt
├── ab-test.json              # Active A/B test configuration
├── candidates/               # Generated prompt variations
│   └── v{timestamp}-{1-5}.txt
└── README.md                 # This file
```

## 🎯 Files

### `blog-generation.txt`
The current production prompt used by `blog-auto-post-v2.py` to generate blog posts.

**When to edit**: Manually tweak the prompt based on learnings, or let the optimizer promote winners automatically.

### `ab-test.json`
Configuration for active A/B tests.

**Structure**:
```json
{
  "test_id": "test_YYYYMMDD",
  "start_date": "2026-04-02T15:30:00",
  "end_date": "2026-04-09T15:30:00",
  "variants": {
    "A": "blog-generation.txt",
    "B": "v20260402-153000-1.txt"
  },
  "results": {
    "A": {"posts": 0, "total_quality": 0, "avg_quality": 0},
    "B": {"posts": 0, "total_quality": 0, "avg_quality": 0}
  },
  "status": "active"
}
```

**Status values**:
- `"active"` - Test is running, blog script will use A/B selection
- `"completed"` - Test finished, winner promoted
- `"inconclusive"` - No clear winner, needs more data

### `candidates/`
Directory containing prompt variations generated by the optimizer.

**Naming convention**: `v{timestamp}-{number}.txt`
- Example: `v20260402-153000-1.txt` (Generated Apr 2, 2026 at 15:30:00, candidate #1)

## 🔄 Workflow

1. **Baseline**: Start with `blog-generation.txt`
2. **Generate**: Run `prompt-optimizer.py optimize` to create candidates
3. **Test**: A/B test runs for 7 days (50/50 split)
4. **Evaluate**: Run `prompt-optimizer.py evaluate` to promote winner
5. **Archive**: Old prompt saved as `blog-generation-{timestamp}-archived.txt`
6. **Repeat**: Continuous improvement cycle

## 📊 Tracking Prompt Performance

Each post in the database has a `prompt_variant` column:
- `"A"` - Generated with variant A (current prompt)
- `"B"` - Generated with variant B (test candidate)
- `"current"` - Generated before A/B testing enabled

Query performance:
```sql
SELECT prompt_variant, COUNT(*) as posts, AVG(quality_score) as avg_quality
FROM posts
GROUP BY prompt_variant;
```

## 🎓 Best Practices

1. **Review candidates** before starting A/B test - AI-generated prompts may need manual refinement
2. **Document changes** - Add notes to this README when you manually edit prompts
3. **Keep archives** - Don't delete old prompts, they contain valuable history
4. **Gradual changes** - Small improvements compound over time
5. **Data-driven** - Let A/B tests run full 7 days before evaluating

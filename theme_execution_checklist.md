# Theme Execution Checklist — Self-Care Cooling Relief Homepage Rebuild

Goal: rebuild the homepage around a single cooling relief product with a clean conversion path, clear trust signals, and zero leftover contamination from prior theme/content clutter.

## P0 — Must fix before launch

- [ ] **Define the single conversion goal**
  - Primary CTA: Buy Now / Shop Now
  - Secondary CTA: Learn More only if it supports purchase intent
  - Remove competing homepage goals, promos, and off-topic links

- [ ] **Rebuild the hero around one clear promise**
  - Headline states the cooling relief benefit in plain language
  - Subheadline explains who it helps and when to use it
  - Product image/GIF shows the product in use, not generic lifestyle filler
  - CTA appears above the fold on desktop and mobile

- [ ] **Lead with the customer problem and relief outcome**
  - Explain the pain point fast: heat, discomfort, soreness, swelling, tension, recovery, etc.
  - Show the desired outcome: cooling, soothing, calming, quick comfort
  - Avoid vague wellness language that hides the actual benefit

- [ ] **Strip out contamination from previous theme/content**
  - Remove irrelevant sections, placeholder copy, stock claims, old badges, old reviews, and generic icons
  - Remove mixed brand voice, inconsistent colors, broken layouts, and duplicate CTAs
  - Remove any content that suggests unrelated products or categories

- [ ] **Create a tight homepage section order**
  1. Hero with value prop + CTA
  2. Proof/trust bar
  3. Benefits block
  4. How it works / how to use
  5. Social proof
  6. Offer / bundle / guarantee
  7. FAQ
  8. Final CTA

- [ ] **Add immediate trust signals near the top**
  - Shipping, guarantee, secure checkout, satisfaction promise
  - If claims are regulated/sensitive, keep wording compliant and non-medical unless substantiated
  - Use trust signals that are specific, not decorative

- [ ] **Clarify the top 3–5 buying reasons**
  - Fast relief / reusable / portable / mess-free / easy to apply / suitable for daily self-care
  - Write these as customer outcomes, not feature jargon

- [ ] **Make mobile conversion friction-free**
  - Short headline, fast image load, visible CTA, stacked sections
  - No giant text walls before the first CTA
  - Keep sticky add-to-cart or sticky CTA if appropriate

- [ ] **Align homepage messaging with product page**
  - Same promise, same offer, same product naming, same trust language
  - No bait-and-switch between homepage and purchase page

## P1 — Strongly recommended for conversion lift

- [ ] **Add a proof bar below the hero**
  - Examples: customer rating, number of orders, satisfaction promise, shipping speed
  - Only use proof you can support

- [ ] **Build a benefits section with scan-friendly structure**
  - 3–6 cards max
  - Each card: short headline + one-sentence explanation
  - Focus on emotional + practical outcomes

- [ ] **Add a simple “how it works” section**
  - Show when to use it
  - Show how long relief typically lasts or how it fits into routine
  - Keep it visually simple and fast to understand

- [ ] **Use real social proof, not generic testimonials**
  - Reviews should mention actual use cases and outcomes
  - Include names/initials, photos, or verified markers where available
  - Prioritize quotes that reduce hesitation

- [ ] **Address objections directly**
  - Is it reusable?
  - Is it comfortable?
  - How cold does it get?
  - Who is it for?
  - How is it different from ice packs / other relief products?

- [ ] **Present the offer clearly**
  - Single unit vs bundle comparison
  - Best-value option highlighted
  - Mention guarantee or risk-reversal close to offer block

- [ ] **Create a comparison section if market is crowded**
  - Product vs ordinary ice pack / messy DIY cooling / short-term alternatives
  - Keep comparison benefit-led and credible

- [ ] **Make imagery instructional, not just aesthetic**
  - Product on body area or use context
  - Show size, texture, fit, portability, and storage/use case
  - Avoid over-relying on abstract spa imagery

- [ ] **Keep the page focused on one audience at a time**
  - If multiple audiences exist, lead with the highest-value one
  - Don’t overload the homepage with too many use cases at once

## P2 — Polishing and optimization

- [ ] **Refine brand voice for self-care + relief**
  - Calm, confident, practical
  - Less hype, more clarity and reassurance

- [ ] **Add routine/lifestyle context**
  - Pre/post workout, end-of-day recovery, hot weather relief, stress decompression, etc.
  - Use cases should support the buying decision, not distract from it

- [ ] **Strengthen FAQ for conversion, not just support**
  - Cover shipping, returns, usage, storage, care, expected experience, gifting
  - Keep answers short and confidence-building

- [ ] **Use repeated CTA moments at natural decision points**
  - After hero
  - After benefits/social proof
  - Near FAQ/final close
  - Keep button text consistent

- [ ] **Check visual hierarchy section by section**
  - Strong headline contrast
  - Readable spacing
  - Obvious CTA styling
  - Consistent iconography and image treatment

- [ ] **Audit every section for a conversion job**
  - If a block does not build desire, trust, clarity, or action, cut it

- [ ] **Prepare a post-launch test list**
  - Hero headline variants
  - Primary image vs use-case image
  - CTA text
  - Bundle framing
  - Social proof placement

## Final acceptance check

- [ ] A first-time visitor understands the product, benefit, and next step within 5 seconds
- [ ] The homepage feels like one coherent brand/product story
- [ ] Every major section reduces friction or increases desire
- [ ] The page is clean on mobile and doesn’t feel “patched together”
- [ ] The final CTA closes with confidence, not clutter
